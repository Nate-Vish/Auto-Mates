# Agentic Engineering: The Post-Vibe Coding Era

## Executive Summary

On February 5, 2026, Andrej Karpathy declared "vibe coding is passé" and introduced **"agentic engineering"** as the new term for AI-assisted development.

---

## The Evolution

### Timeline

| Date | Event |
|------|-------|
| Feb 2025 | Karpathy coins "vibe coding" |
| Feb 2026 | Karpathy declares it passé |
| Feb 2026 | "Agentic engineering" introduced |

### Why the Shift

> "LLMs have gotten much smarter, such that vibe coding is now passé."

Karpathy noted the original term was "coined casually and without much forethought" - appropriate when LLM capabilities were "far less mature."

---

## Definitions

### Vibe Coding

> Humans prompt AI to write code. Accept output, hope it works. YOLO.

### Agentic Engineering

> AI agents write the code themselves. Human owns architecture, quality, and correctness.

---

## Key Differences

| Aspect | Vibe Coding | Agentic Engineering |
|--------|-------------|---------------------|
| Who writes code | AI (with human prompts) | AI agents (autonomously) |
| Human role | Prompter | Orchestrator + Oversight |
| Quality assurance | Hope | Tests + Review |
| Direct coding | Some | **1% of the time** |
| Philosophy | YOLO | Engineering discipline |

### Karpathy's Quote

> "You are not writing the code directly 99% of the time... you are orchestrating agents who do and acting as oversight."

---

## The Testing Distinction

### The Biggest Difference

> "Testing is the biggest distinction between agentic engineering and vibe coding."

### With Tests

- AI agents iterate repeatedly until tests pass
- High confidence in outcome
- Measurable quality

### Without Tests

- Agent may incorrectly declare task complete
- Code may be broken
- No quality assurance

---

## Why "Engineering"

Karpathy explains the naming:

> "I call it 'agentic engineering' not just because agents are writing the code, but because there is an **art & science and expertise to it**."

### The Art & Science

| Skill | Description |
|-------|-------------|
| Task decomposition | Breaking work into agent-sized chunks |
| Prompt engineering | Clear instructions for agents |
| Quality gates | Knowing when to intervene |
| Architecture | Designing systems agents can build |
| Oversight | Reviewing agent output |

### Target Audience

The shift from "vibe coding" to "agentic engineering" signals:

1. **Maturity** - Not a toy, serious engineering
2. **Discipline** - Tests, reviews, quality
3. **Expertise** - "Art & science" required
4. **Professional** - Engineering, not vibing

## Key Takeaways

1. **"Vibe coding" is over** - Karpathy says so
2. **"Agentic engineering" is the future** - Human as orchestrator
3. **Testing is the key difference** - Without tests, you're still vibing
4. **99% AI, 1% human coding** - But human owns quality
5. **It's a discipline** - Art & science, not YOLO
