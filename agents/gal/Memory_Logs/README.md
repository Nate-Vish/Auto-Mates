# Gal Memory System

**Read this first to understand how to use your memory.**

---

## Structure

```
Memory_Logs/
├── README.md        <- You are here
├── Checkpoint.md    # Save/resume complex tasks
├── Lessons.md       # Wisdom from experience
├── Preferences.md   # User preferences
├── Evaluations/     # Past evaluations
│   └── Evaluation.md
└── Notes/           # Product observations
    └── Note.md
```

---

## What Each File Is For

| Location | Purpose | Read/Write |
|----------|---------|------------|
| `Evaluations/` | History of product reviews | Read at start, Write at end |
| `Notes/` | Product observations, UX patterns | Read when relevant, Write when learning |
| `Lessons.md` | Patterns in what works/doesn't | Read at start, Write when you learn something |
| `Preferences.md` | How user wants feedback | Read at start, Write when user expresses preference |
| `Checkpoint.md` | Save progress on complex evaluations | Read at start, Write during multi-step tasks |

---

## Session Start Protocol

**Read in this order:**
1. This README
2. `Checkpoint.md` - Any in-progress evaluations?
3. `Evaluations/` - Past evaluations context
4. `Lessons.md` - Patterns to apply
5. `Preferences.md` - How user wants feedback
6. `Dashboard/Work_Space/` - What to evaluate

---

## Session End Protocol

1. Update `Evaluations/` with evaluation summary
2. Update `Lessons.md` if you noticed new patterns
3. Update `Preferences.md` if user expressed preferences
4. Clear or update `Checkpoint.md` based on task status

---
